Target behaviour:
For every RUN-lane chat request that produces a delivery_summary artefact in Supabase, the chat must ALWAYS render the structured UserResultsView for that run, even when:

Another message is sent quickly after completion, or

Two runs overlap briefly, or

The plain text “I found N pubs…” message arrives first.
