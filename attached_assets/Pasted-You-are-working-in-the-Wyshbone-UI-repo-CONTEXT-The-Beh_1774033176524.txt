You are working in the Wyshbone UI repo.

CONTEXT:
The Behaviour Judge (BJ) fires in Tower after judging artefacts. It fires for BOTH per-loop `final_delivery` artefacts AND the `combined_delivery` artefact (which contains ALL leads from ALL loops merged together). Each firing persists a row to the `behaviour_judge_results` table in Supabase.

BUG:
When a multi-loop run produces a `combined_delivery` artefact (e.g. 8 leads from 2 loops), the UI shows the BJ verdict from the last `final_delivery` (e.g. "2 delivered") instead of the `combined_delivery` verdict (e.g. "8 delivered").

ROOT CAUSE:
In `server/supabase-client.ts`, the function `getBehaviourJudgeResult(runId)` queries `behaviour_judge_results` with `.eq('run_id', runId).maybeSingle()`. If there are multiple BJ rows for the same run_id (one for each judged artefact type), it gets whichever one Supabase returns — which is the wrong one.
