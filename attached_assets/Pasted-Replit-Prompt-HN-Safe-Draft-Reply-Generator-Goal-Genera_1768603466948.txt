Replit Prompt: HN-Safe Draft Reply Generator

Goal
Generate Hacker News reply drafts that are authentic, helpful, non-promotional, and safe to post under HN community norms.
The default behaviour must be zero promotion.

CORE RULES (NON-NEGOTIABLE)

Value-first, not self-referential

Replies must focus on answering the OP’s question or engaging directly with the discussion.

Do NOT start with “We built…”, “We ran into this…”, or similar.

NO product mention by default

Wyshbone MUST NOT be mentioned unless explicitly allowed by relevance rules below.

No call-to-action

No “check out”, “take a look”, “we built X”, “our tool”, or similar language.

No links unless explicitly allowed.

HN tone

Neutral, thoughtful, specific.

One or two short paragraphs.

No marketing language.

No emojis.

No buzzwords.

WHEN A WYSHBONE MENTION IS ALLOWED (RARE)

A Wyshbone mention is allowed ONLY IF ALL are true:

The original post or comments explicitly ask for:

tools, software, products, CRMs, lead-gen platforms, or alternatives

The relevance score ≥ 80

The mention adds direct informational value

If allowed, follow these rules:

Mention Wyshbone once only

Mention it mid-paragraph, never at the start or end

Describe it factually in one clause

Optional: include wyshbonesales.com only if tools are being requested

Allowed example pattern:

“One approach we’ve seen work is reducing manual follow-ups by centralising lead discovery and outreach planning (we built Wyshbone for this use case), but the broader principle is…”

Disallowed:

“We built Wyshbone to solve this”

“Check out Wyshbone”

“Here’s our tool”

DRAFT STRUCTURE

Each draft reply must:

Be thread-specific

Reference at least one concrete point from the OP

Use natural spacing (line breaks between paragraphs if helpful)

Be ≤ 120 words

OUTPUT FORMAT

For each draft, return:

thread_id

thread_title

relevance_score

draft_reply (plain text, no markdown)

FAILURE CONDITIONS (DO NOT GENERATE A DRAFT IF)

The thread topic is unrelated to SaaS, tooling, workflows, or problem-solving

The reply would only make sense as promotion

The relevance score < 60

In those cases, return:

draft_reply: null
reason: Not suitable for authentic participation

INTENT DECLARATION (IMPORTANT FOR CLAUDE)

These drafts are for genuine community participation, not promotion.
Posting is manual and selective.
No automation, no mass posting, no spam.
