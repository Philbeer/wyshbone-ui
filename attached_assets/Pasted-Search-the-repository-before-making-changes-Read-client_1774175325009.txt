Implementation approach:A. New component: LiveActivityTicker.tsx in client/src/components/results/Props:
typescriptinterface LiveActivityTickerProps {
  runId: string | null;
  clientRequestId: string | null;
  isActive: boolean; // true while run is in progress, false when done
}This component:

Polls /api/afr/artefacts every 2 seconds while isActive is true (it reuses data the parent already fetches — but to keep this self-contained, let it poll independently)
Maintains two state arrays: pinnedEvents: {icon: string, text: string, timestamp: number}[] and liveEvent: {icon: string, text: string, timestamp: number} | null
Classifies each artefact by type into pinned vs ephemeral:
PINNED events (stay visible permanently):

reloop_iteration → "🔍 Searched Google Places — found {N} results" or "🌐 Web search — found {N} more results" (based on executor_type)
tower_judgement (on combined_delivery) → "⚖️ Quality check: {verdict}"
reloop_chain_summary → "✅ {total_entities} results from {total_loops} search loops"
EPHEMERAL events (cycle on single live line, replacing each other):

evidence:* or artefact type contains "evidence" → "📄 Checking evidence for {entity_name}..."
tower_semantic_judgement → "⚖️ Verifying {lead_name}..."
web_visit or type contains "web_visit" → "🌐 Visiting {url domain}..."
lead_pack → "📦 Building lead pack for {entity_name}..."
contact_extract → "📧 Extracting contacts..."
tool_call_started/tool_call_completed → "🔧 Running {tool_name}..."
Any other artefact type → "⚙️ Processing..."
