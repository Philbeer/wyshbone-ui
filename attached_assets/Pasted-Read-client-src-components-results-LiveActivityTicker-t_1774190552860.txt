Read client/src/components/results/LiveActivityTicker.tsx. Fix the intent narrative timing bug.

PROBLEM: The ticker self-fetches artefacts using client_request_id and runId from props. But the intent_narrative artefact is stored under the SUPERVISOR's run_id, not the UI's agent_run id. The supervisor_run_id bridge in agent_runs takes time to be set up. During that gap, the artefact fetch returns rows that don't include intent_narrative because it's querying the wrong run_id.

FIX: The stream response (/api/afr/stream) already returns a run_id field. We also need to capture the supervisor_run_id if available. Use these as additional lookup keys for the artefact fetch.

    event_count: number;
