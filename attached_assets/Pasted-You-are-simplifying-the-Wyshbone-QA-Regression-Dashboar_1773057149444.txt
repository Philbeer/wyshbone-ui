DEFINITIONS
-----------------------------------

SYSTEM RELIABILITY

This answers:
"Did the machinery run?"

Use:
Working = Healthy + Degraded

Formula:
System Reliability = Working / Total

Timeout counts against reliability because the user experiences timeout as failure.

Do NOT show:
- Clean Run Rate
- System Working Rate
- separate Healthy vs Degraded headline percentages

You may still keep Healthy / Degraded internally if needed, but do not surface them as primary metrics.

Instead:
collapse Healthy + Degraded into a single visible count called Working.

BROKEN means actual broken run / invalid pipeline state.

TIMEOUT means benchmark did not complete.

-----------------------------------

AGENT PERFORMANCE

This answers:
"Did the agent succeed at the task?"

Use visible counts:
- Pass
- Partial
- Fail
- Unknown

Formula:
Agent Success Rate = Pass / (Pass + Partial + Fail)

Unknown is excluded because the run did not complete enough to judge quality.

Do NOT show:
- N/A as a primary visible category

If N/A exists internally, map it into Pass or Unknown depending on the benchmark expectation, or keep it hidden unless absolutely necessary.
