Read server/routes/afr.ts. Fix the POST /api/afr/artefacts handler to auto-set the supervisor_run_id bridge.

ROOT CAUSE OF INTENT DELAY: The supervisor POSTs artefacts with BOTH runId (its own internal ID) and clientRequestId. But the POST handler ignores clientRequestId — it only uses runId to store the artefact. The bridge (supervisor_run_id on agent_runs) is set up later by a separate /run-bridge call. Until that call happens (~8-12s), the GET handler can't find supervisor artefacts.

FIX: In the POST handler, when clientRequestId is present and runId differs from the agent_run's id, auto-link the supervisor_run_id bridge. This sets it up on the FIRST artefact POST (which is intent_narrative itself), so the GET handler finds it immediately on the next poll.

Find the POST "/artefacts" handler. After the artefact is inserted (after the RETURNING id line and the console.log), add this block BEFORE the res.status(201) line:

      // ── Auto-link supervisor_run_id bridge if clientRequestId is present ──
      // The supervisor sends its own runId + the shared clientRequestId.
      // By linking early, the GET handler can find supervisor artefacts immediately.
      if (clientRequestId && runId) {
        try {
          const agentRunForBridge = await storage.getAgentRunByClientRequestId(clientRequestId);
          if (agentRunForBridge && agentRunForBridge.id !== runId && !(agentRunForBridge as any).supervisorRunId) {
            await db.execute(
              sql`UPDATE agent_runs SET supervisor_run_id = ${runId}, updated_at = ${Date.now()} WHERE id = ${agentRunForBridge.id} AND supervisor_run_id IS NULL`
            );
            console.log(`🔗 [AFR artefacts POST] Auto-bridged supervisor_run_id=${runId} → agent_run=${agentRunForBridge.id} (via crid=${clientRequestId.slice(0, 12)})`);
          }
        } catch (bridgeErr: any) {
          // Non-fatal — bridge will be set by /run-bridge eventually
          console.warn(`[AFR artefacts POST] Auto-bridge failed:`, bridgeErr.message);
        }
      }
